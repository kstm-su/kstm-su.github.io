# kstm 光芒祭CTF 2025 回答速報

本日は信州大学工学部光芒祭にお越しくださりありがとうございました!

今回の開催では、延べ42のグループの皆様に参加していただきました。私達のブースまで足を運んでいただき、ありがとうございます。

この記事では、kstmの出展のCTFに登場した問題について、簡単な解説をします。

## 問題解説

### 【Easy】機関車を走らせろ
この問題は、Unix系OSのジョークコマンド、[slコマンド](https://github.com/mtoyoda/sl)を改造した問題です。

ターミナル上にてslコマンドを利用して、表示される機関車にフラッグが表示されるようになっています。


### 【Easy】灯台下暗し
この問題は、JavaScriptの"blur"イベントを利用した問題です。

イベントというのは、ユーザーの操作や、ダウンロードの完了など、いつ来るかわからない出来事が発生したあとにプログラムを実行するという仕組みです。

blurイベントは、ブラウザのタブが非アクティブになったときに発生するイベントです。これに呼応させて、タブのタイトルを動的に変更するようにしています。これを利用して、ヒントページを新しいタブで開くことで、問題ページのタブタイトルがフラッグになるようになっています。

### 【Easy】暗号化メール事件
この問題は、シーザー暗号をテーマとした問題です。


シーザー暗号は、アルファベットを指定の回数"ずらす"ことで暗号化する手法です。問題では回転数は13と指定されていたので、x→kになるように13個ずらすと解答が得られました。

問題文に与えられていたメール
```
Uryyb, V'z gur frperg cebwrpg bs bhe sbyybjvat:
xfgz{PGS_jvgu_ebg13!}
```
は復号することで、
```
Hello, I'm the secret project of our following: 
kstm{CTF_with_rot13!}
```
となります。

### 【Medium】アノニマスの受難
この問題は、
- ブルートフォース攻撃
- IDOR

この２つの攻撃方法を駆使することを主題とした問題です。

まず、ブルートフォース攻撃を行うJavaScriptのファイルが添付されています。これをログインページで実行することで、総当たりでパスワードの突破を試みて量子コーヒー研究部の保管システムに入ります。

次にURLの
```
mypage?user-id=qcrd
```
のid部分を"ukkr"に変更することで、「ものつくらないサークルukkr」の資料保管システムに入り込め、フラッグを得ることが出来ます。

現実のサイトではこんな攻撃が用意に成立しないように設計するのが普通です。問題を解けるように、わざと脆弱なサイトを構築していました。

### 【Medium】北陸のロマン
この問題は、jpegファイルに保存されたExifデータに何が保存されているかを主題とした問題です。

どの電車に乗っていたかを特定するには画像の撮影時刻を知ることが重要です。しかし、この画像には時計やその類のものは写っていません。そこで、利用するのがjpegにあるExifにある撮影時刻のデータです。画像をダウンロードして、このデータを見ると、撮影時刻が
```
2025/09/28 13:21
```
とわかります。

あとは[Yahoo!路線情報](https://transit.yahoo.co.jp/search/result?from=%E5%A4%A7%E9%98%AA&to=%E9%95%B7%E9%87%8E&fromgid=&togid=&flatlon=&tlatlon=&via=%E6%95%A6%E8%B3%80&viacode=&y=2025&m=09&d=28&hh=13&m1=3&m2=0&type=4&ticket=ic&expkind=1&userpass=1&ws=1&s=0&al=1&shin=1&ex=1&hb=0&lb=1&sr=1)などでこの時刻に長野駅に到着するように乗り換え検索を行うと、乗車した列車が、はくたか560号とサンダーバード11号であることがわかります。

ところで、文章中にて「列車番号」と表記したことで、答えを4011と誤解させてしまった事例が発生しました。「号数」というのが正しかったですね。

### 【Hard】消えたデータを追え！
この問題は、Webフロントエンドの基本的な技術（HTML, CSS, JS）に関する知識を問い、開発者ツールをいかに使いこなせるかを試すものです。フラグは3つのパーツに分割され、それぞれ異なる場所に隠されています。

#### ステップ1: ヒントを下にした取っ掛かり

まず、問題文のヒントにブラウザの開発者ツールを使用が不可欠と書かれています。

#### ステップ2: コンソールの確認 - 第1のキー

指示通りに開発者ツールのコンソールを開くと、`script.js` によって出力された以下のメッセージが表示されています。

```
最初のキーのかけら: ZmxhZ3t3ZWJf
ヒント: この文字列は特殊な方法でエンコードされているようだ。'atob()'関数が役に立つかもしれない。
```

ここで第1のキーのかけら `ZmxhZ3t3ZWJf` を入手できます。ヒントにある通り、これはBase64でエンコードされた文字列です。コンソールで `atob()` 関数を使い、デコードします。

```javascript
atob('ZmxhZ3t3ZWJf')
// 実行結果: "flag{web_"
```

これにより、フラグの先頭部分 **`flag{web_`** を得ることができます。

#### ステップ3: CSSの調査 - 第2のキー

次に、HTMLやCSSに隠された情報を探します。`index.html` には `.secret-message` というクラス名を持つpタグが存在します。

```html
<p class="secret-message">見た目だけが全てではない。</p>
```

しかし、このテキストはブラウザ上では見えません。`style.css` を確認すると、その理由がわかります。

```css
.secret-message {
    color: #16213e; /* 背景色と同じにして見えなくしている */
}

.secret-message::after {
    content: "X2tleV9pc19k"; /* キーの一部 */
}
```

テキストの色が背景色と同じに設定されているため、不可視になっていました。さらに重要なのは、`::after` 擬似要素の `content` プロパティに、第2のキーのかけら **`X2tleV9pc19k`** が設定されていることです。

これも同様にBase64デコードを行います。

```javascript
atob('X2tleV9pc19k')
// 実行結果: "_key_is_d"
```

これにより、フラグの中間部分 **`_key_is_d`** を得ることができます。

#### ステップ4: JavaScriptの解析 - 第3のキー

最後に `script.js` を詳細に調査します。ファイルの下部に、どこからも呼び出されていない（未使用の）関数 `getThirdKeyPart()` が存在します。

```javascript
// 誰も気づかない関数...
// この関数はどこからも呼び出されていない。
function getThirdKeyPart() {
    // 16進数から文字列へ (※コメントはミスリード。実際はBase64)
    const hex = 'MzA2ZWN9';
    return hex;
}
```

このような未使用の関数は、CTFでは意図的に隠されたロジックであることが多いです。コンソールからこの関数を手動で実行します。

```javascript
getThirdKeyPart()
// 実行結果: "MzA2ZWN9"
```

これで第3のキーのかけら **`MzA2ZWN9`** を入手できました。これもBase64デコードします。

```javascript
atob('MzA2ZWN9')
// 実行結果: "306ec}"
```

これにより、フラグの末尾部分 **`306ec}`** が得られます。


#### ステップ5: フラグの組み立て

ステップ2、3、4で入手した3つのパーツを発見した順に結合します。

1.  `flag{web_`
2.  `_key_is_d`
3.  `306ec}`

結合した結果、最終的なフラグが完成します。

```
flag{web__key_is_d306ec}
```
